Semantic Extraction — Natural Language to Logical Form
LLM (Gemini)
▶
The LLM receives the user's natural language along with the full tag vocabulary and synonym list injected into its system prompt. Its only job: emit a structured function call containing a recursive JSON logical form. It never routes, never scores, never queries the database.
// LLM emits a "plan" function call with these arguments: { "origin": "Siam", "goal": { "operator": "and", "args": [ { "operator": "any_tag", "tags": ["night_market"] }, { "operator": "any_tag", "tags": ["street_food"] }, { "operator": "none_tag", "tags": ["crowded"] } ] }, "time_context": { "weekday": "sat", "hour": 20, "minute": 0 } }
This JSON tree is the Abstract Syntax Tree (AST) of the user's intent. Think of it like a parse tree: the LLM is the parser, and Prolog is the evaluator.

Candidate Evaluation — The Meta-Interpreter Filters POIs
Prolog (rules.pl — satisfies/3 + candidates/3)
▶
candidates/3 iterates over every POI in the knowledge base and calls satisfies(Poi, Goal, t(sat,20,0)). The meta-interpreter walks the AST recursively: for AND, every child must succeed; for any_tag, at least one tag must be present (respecting subsumption + temporal gating); for none_tag, no tag may be present.

At Saturday 8 PM, time_matches(after_sunset, t(sat, 20, 0)) succeeds because 20 ≥ 18. So both night_market and high_density are active on Jodd Fairs right now.

Constraint Relaxation — Finding the Minimal Drop Set
Prolog (rules.pl — relax/4)
▶
relax/4 uses select/3 to systematically try removing one conjunct at a time from the AND list, then re-runs candidates/3 on the surviving sub-goal. It stops at the first minimal drop that produces a non-empty candidate set. This is bounded: single drops first, then pairs, max 2 drops total.

C = ['Chinatown'-... ] % Non-empty! But Prolog tries ALL single drops via backtracking... % Try 2: Drop any_tag([street_food]) ?- candidates(and([any_tag([night_market]), none_tag([high_density])]), t(sat,20,0), C). C = ['Asiatique'-... ] % Asiatique is night_market + NOT high_density at 8 PM % Try 3: Drop none_tag([high_density]) ← THIS IS THE KEY ONE ?- candidates(and([any_tag([night_market]), any_tag([street_food])]), t(sat,20,0), C). C = ['Jodd Fairs Night Market'-'Phra Ram 9 (BL20)'-0] ✓ Non-empty! % First single-drop solution found (via select/3 backtracking): Dropped = [none_tag([high_density])] Survivors = ['Jodd Fairs Night Market'-'Phra Ram 9 (BL20)'-0]
The orchestrator now knows: "I had to drop your 'not crowded' constraint to find anything. The best match is Jodd Fairs, but it will be crowded at 8 PM." This is surfaced as a relaxation_note in the response.

Route Audit — Prolog Checks Python's Work

▶
